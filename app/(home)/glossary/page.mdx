| Availability      	| The number of full copies of a file (or set of files and directories) directly available to the client. Each seed adds 1.0 to this number, as they have one complete copy of the file. A connected peer with a fraction of the file available adds that fraction to the availability, if no other peer has this part of the file.                                                                                                                                                                                                                                                         	|
| BitTorrent        	| The most widely used form of peer-to-peer filesharing. BitTorrent is a decentralised system which coodinates download distribution (see "Tracker") but doesn't host any downloads. Downloads are automatically segregated into pieces which are gathered from peers and re-assembled into usable files by the torrent client.                                                                                                                                                                                                                                                             	|
| Connectability    	| Connectability relates to the state of 'being connectable', meaning an incoming port on your computer is open, allowing other peers in the swarm to connect to you in order to share data. A connectable user can share data with anyone in the swarm, but an unconnectable user can't and is restricted to only those with open ports (More Info). Being 'unconnectable' also negatively affects the performance of the swarm and in some cases the speed of your download/upload.                                                                                                       	|
| Cross seed        	| When the same torrent is on two different trackers, you can donwload it from one, then seed to both.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      	|

| Hash              	| The hash is a digital fingerprint in the form of a string of alphanumeric characters (typically hexadecimal) in the .torrent file that the client uses to verify the integrity of data that is being transferred. "Hash" is the shorter form of the word "hashsum". Torrent files contain information such as the URL of the torrent's tracker, a list of files contained within a torrent, their size, format, and relevant attributes. The torrent client will check each "piece" of the torrent against the hash. If it fails verification, the data is discarded and requested again. 	|
| Hit-and-run (HnR) 	| To HnR means to intentionally leech a torrent. If you HnR on Private Trackers, you will likely be penalized.                                                                                                                                                                                                                                                                                                                                                                                                                                                                              	|
| Leech             	| The more accurate definition of Leech is a BitTorrent user that has a negative effect on the swarm, usually by performing a HnR.
